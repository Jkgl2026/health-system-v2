# 部署指南

## 前置准备

1. 已注册微信小程序账号
2. 已获取小程序 AppID
3. 已完成小程序基本信息填写
4. 已配置服务器域名（如有需要）

## 开发阶段

### 1. 开发环境搭建

```bash
# 克隆项目
git clone <repository-url>
cd miniprogram-code

# 安装依赖
npm install

# 启动开发服务器
npm run dev:weapp
```

### 2. 微信开发者工具配置

1. 打开微信开发者工具
2. 导入项目，选择 `dist` 目录
3. 填写项目信息：
   - 项目名称：健康自检小程序
   - AppID：你的小程序 AppID
   - 开发模式：小程序
   - 后端服务：不使用云服务

### 3. 测试功能

在开发者工具中测试所有功能：
- ✅ 自检功能
- ✅ 结果展示
- ✅ 数据分析
- ✅ 微信登录
- ✅ 数据导出/导入
- ✅ 页面跳转

## 预览和调试

### 1. 真机预览

1. 在开发者工具中点击「预览」
2. 使用微信扫描二维码
3. 在手机上查看和测试

### 2. 真机调试

1. 在开发者工具中点击「真机调试」
2. 使用微信扫描二维码
3. 在手机上打开调试模式
4. 查看控制台日志和网络请求

### 3. 体验版上传

1. 在开发者工具中点击「上传」
2. 填写版本号和备注
3. 上传成功后在微信公众平台查看

## 发布流程

### 1. 版本审核前检查

- [ ] 完整测试所有功能
- [ ] 修复所有已知 Bug
- [ ] 检查页面布局和样式
- [ ] 确认文案准确无误
- [ ] 检查用户隐私保护
- [ ] 确认符合微信小程序规范

### 2. 构建生产版本

```bash
# 构建生产版本
npm run build:weapp
```

构建完成后，`dist` 目录包含生产版本的代码。

### 3. 提交审核

1. 登录 [微信公众平台](https://mp.weixin.qq.com/)
2. 进入「版本管理」->「开发版本」
3. 选择刚才上传的版本
4. 点击「提交审核」
5. 填写审核信息：
   - 功能页面描述
   - 功能截图
   - 补充说明

### 4. 审核时间

- 正常审核时间：1-7 个工作日
- 加急审核：小程序已上线后可申请加急

### 5. 审核通过

审核通过后，点击「全量发布」即可正式上线。

## 版本管理

### 版本号规范

采用语义化版本号 (Semantic Versioning)：

```
主版本号.次版本号.修订号

例如：1.0.0
- 1：主版本号（重大更新）
- 0：次版本号（新增功能）
- 0：修订号（Bug 修复）
```

### 版本更新流程

1. 更新版本号：修改 `project.config.json` 和 `CHANGELOG.md`
2. 构建新版本：`npm run build:weapp`
3. 上传新版本：在开发者工具中点击「上传」
4. 提交审核：在微信公众平台提交审核
5. 发布：审核通过后发布

## 常见问题

### 1. 审核被拒

**常见原因**：
- 功能不完整或无法使用
- 违反微信小程序运营规范
- 用户信息收集未获得授权
- 页面内容违规

**解决方案**：
- 查看审核反馈
- 修复问题后重新提交
- 联系微信客服咨询

### 2. 用户登录失败

**可能原因**：
- AppID 配置错误
- 未配置用户隐私保护指引
- 授权域名未配置

**解决方案**：
- 检查 AppID 是否正确
- 在微信公众平台配置用户隐私保护指引
- 检查授权域名设置

### 3. 文件导入导出失败

**可能原因**：
- 微信开发者工具模拟器限制
- 文件系统权限问题
- 文件格式错误

**解决方案**：
- 使用真机调试
- 检查文件格式是否为 JSON
- 确保文件内容格式正确

### 4. 页面加载缓慢

**可能原因**：
- 数据量过大
- 图片未压缩
- 代码未优化

**解决方案**：
- 优化数据加载（分页、懒加载）
- 压缩图片资源
- 优化代码，减少不必要的计算

## 性能优化

### 1. 代码优化

- 减少不必要的组件渲染
- 使用 useMemo 和 useCallback
- 优化事件处理函数
- 避免深层嵌套的组件

### 2. 资源优化

- 压缩图片资源
- 使用 CDN 加载静态资源
- 合理使用本地缓存
- 减少网络请求

### 3. 用户体验优化

- 添加加载状态提示
- 优化页面过渡动画
- 提供错误提示和重试机制
- 优化触摸响应

## 安全建议

### 1. 数据安全

- 用户信息加密存储
- 敏感数据使用 HTTPS 传输
- 定期备份数据
- 实现数据恢复机制

### 2. 隐私保护

- 明确告知用户数据用途
- 获得用户授权后再使用数据
- 提供数据导出和删除功能
- 遵守相关法律法规

### 3. 代码安全

- 不在前端存储敏感信息
- 验证用户输入
- 防止 XSS 攻击
- 定期更新依赖包

## 监控和维护

### 1. 错误监控
