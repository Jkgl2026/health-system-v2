# 管理后台首页测试步骤（简洁版）

## 一、访问管理后台首页

### 方法1：登录后自动跳转
1. 访问登录页：`pages/admin/login/index`
2. 输入账号：`admin`
3. 输入密码：`admin123`
4. 点击「登录」
5. 验证：自动跳转到管理后台首页

### 方法2：直接访问
1. 在开发者工具「页面路径」输入：`pages/admin/dashboard/index`
2. 点击「编译」

---

## 二、权限控制测试

### 测试1：已登录访问
1. 确保 `admin_token` 存在于 Storage
2. 访问管理后台首页
3. 验证：页面正常显示，无跳转

### 测试2：未登录访问
1. 清除 Storage 中的 `admin_token`
2. 访问管理后台首页
3. 验证：
   - 显示「请先登录」提示
   - 1秒后跳转到登录页

---

## 三、数据统计测试

### 测试1：检查统计卡片
验证以下4个卡片显示正确：
- 👥 用户数
- 📋 自检次数
- 📊 平均分
- ⚠️ 高风险用户（红色边框）

### 测试2：验证数据准确性
1. 在用户端完成5次自检
2. 检查「自检次数」是否为5
3. 检查「平均分」计算是否正确
4. 检查「高风险用户」是否正确

---

## 四、健康等级分布图表测试

### 测试1：检查图表显示
1. 滚动到「健康等级分布」区域
2. 验证：
   - 显示环形图
   - 中心显示「总人数」
   - 各等级用不同颜色表示

### 测试2：验证颜色
- 优秀：绿色
- 良好：蓝色
- 一般：黄色
- 需要关注：橙色
- 建议就医：红色

### 测试3：测试刷新
1. 完成一次新的自检
2. 点击右上角 🔄 按钮
3. 验证：图表自动更新

---

## 六、登录日志测试

### 测试1：检查日志列表
1. 滚动到「近期登录日志」
2. 验证：
   - 显示最近10次登录
   - 每条显示图标和时间
   - 时间格式正确（刚刚/X分钟前/X小时前/X天前）

### 测试2：测试日志更新
1. 退出登录
2. 重新登录
3. 返回管理后台首页
4. 验证：最新日志显示「刚刚」
