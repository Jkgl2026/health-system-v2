# Coze 平台数据导入指南

## 📊 问题说明

### 为什么看不到数据？

你在 Coze 平台（https://cd776816-213e-4c84-af60-dbe5b397d82e.dev.coze.site/admin/login）中看不到数据，是因为：

- **本地数据库**（localhost:5000）有59个用户
- **Coze 平台数据库**是独立的数据库实例
- 两个数据库是分开的，不会自动同步

---

## 🔧 解决方案

### 方案1：将本地数据导入到 Coze 平台数据库

**前提条件：**
- 需要获取 Coze 平台的数据库连接信息
- 数据库类型：Supabase PostgreSQL

**步骤：**

1. **获取数据库连接信息**
   - 登录 Coze 平台：https://www.coze.com/
   - 进入你的项目：健康自检系统
   - 查看环境变量配置
   - 找到数据库连接字符串

2. **使用数据导入工具**
   - 我们已经导出了数据：`assets/export-data-20260203-220409.json`
   - 需要创建导入脚本
   - 执行数据导入

---

### 方案2：直接在 Coze 平台数据库中操作

**前提条件：**
- 有 Coze 平台的数据库访问权限
- 可以直接操作 Supabase 数据库

**步骤：**

1. **登录 Supabase Dashboard**
   - 获取 Supabase 项目 URL
   - 使用邮箱和密码登录

2. **打开 SQL Editor**
   - 执行 SQL 插入数据

---

### 方案3：使用 API 导入（推荐）

**步骤：**

1. **创建数据导入 API**
   - 使用我们导出的 JSON 文件
   - 通过 API 导入到 Coze 平台

2. **执行导入**
   - 调用导入 API
   - 验证导入结果

---

## 💡 替代方案

如果无法获取 Coze 平台数据库连接信息，可以：

### 选项1：在 Coze 平台重新创建数据
1. 通过健康自检系统添加用户
2. 数据会自动保存到 Coze 平台数据库

### 选项2：使用本地环境
1. 继续使用本地环境（localhost:5000）
2. 数据已保留在本地数据库

### 选项3：配置数据库连接
1. 将 Coze 平台数据库连接到本地
2. 这样两个环境可以共享数据
