## 本地开发环境搭建

### 步骤1：安装Node.js

1. 访问 https://nodejs.org/
2. 下载并安装 LTS 版本（推荐 v24.x）
3. 验证安装：
```bash
node --version
# 应显示：v24.x.x
```

### 步骤2：安装PostgreSQL

1. 访问 https://www.postgresql.org/download/
2. 下载并安装 PostgreSQL 14.x 或更高版本
3. 安装完成后，设置默认密码（记住此密码）
4. 验证安装：
```bash
psql --version
# 应显示：psql (PostgreSQL) 14.x
```

### 步骤3：安装pnpm

```bash
npm install -g pnpm
pnpm --version
# 应显示：8.x.x
```

### 步骤4：克隆/下载项目代码

```bash
cd /path/to/workspace
# 将项目代码复制到此目录
```

### 步骤5：安装项目依赖

```bash
cd /path/to/workspace
pnpm install
```

### 步骤6：配置数据库连接

编辑 `src/lib/db.ts` 文件：

```typescript
const pool = new Pool({
  host: process.env.DB_HOST || 'localhost',
  port: parseInt(process.env.DB_PORT || '5432'),
  database: process.env.DB_NAME || 'health_check_db',
  user: process.env.DB_USER || 'postgres',
  password: process.env.DB_PASSWORD || 'your_password',
});
```

### 步骤7：创建数据库并恢复数据

```bash
# 创建数据库
createdb health_check_db

# 恢复数据库结构
psql health_check_db < health_check_db_schema.sql

# 恢复数据
psql health_check_db < health_check_db_data.sql
```

### 步骤8：启动开发服务器

```bash
pnpm run dev
```

服务器将在 http://localhost:5000 启动

### 步骤9：访问系统

1. 打开浏览器
2. 访问 http://localhost:5000
3. 使用默认账号登录：
   - 用户名：admin
   - 密码：123456

## 生产环境部署

### 方案1：使用 PM2 部署（推荐）

确保服务器已安装：
- Node.js 24+

### 方案2：使用 Docker 部署

#### 创建 Dockerfile

```dockerfile
FROM node:24-alpine AS base

# Install dependencies only when needed
FROM base AS deps
RUN apk add --no-cache libc6-compat
WORKDIR /app

COPY package.json pnpm-lock.yaml* ./
RUN npm install -g pnpm
RUN pnpm install --frozen-lockfile

# Rebuild the source code only when needed
FROM base AS builder
WORKDIR /app
COPY --from=deps /app/node_modules ./node_modules
COPY . .

RUN npm install -g pnpm
RUN pnpm run build

# Production image
FROM base AS runner
WORKDIR /app

ENV NODE_ENV production

RUN addgroup --system --gid 1001 nodejs
RUN adduser --system --uid 1001 nextjs

COPY --from=builder /app/public ./public
COPY --from=builder --chown=nextjs:nodejs /app/.next/standalone ./
COPY --from=builder --chown=nextjs:nodejs /app/.next/static ./.next/static

USER nextjs

EXPOSE 5000

ENV PORT 5000
ENV HOSTNAME "0.0.0.0"

CMD ["node", "server.js"]
```

#### 创建 docker-compose.yml

```yaml
version: '3.8'

services:
  postgres:
    image: postgres:14-alpine
    environment:
      POSTGRES_DB: health_check_db
      POSTGRES_USER: postgres
      POSTGRES_PASSWORD: your_password
    volumes:
      - postgres_data:/var/lib/postgresql/data
    ports:
      - "5432:5432"

  app:
    build: .
    ports:
      - "5000:5000"
    environment:
      DB_HOST: postgres
      DB_PORT: 5432
      DB_NAME: health_check_db
      DB_USER: postgres
      DB_PASSWORD: your_password
    depends_on:
      - postgres
    restart: always

volumes:
  postgres_data:
```

#### 启动服务

```bash
docker-compose up -d
```

## Nginx配置

### 基础配置

创建 `/etc/nginx/sites-available/health-check-admin`：

```nginx
upstream health_check_backend {
    server 127.0.0.1:5000;
    keepalive 64;
}

server {
    listen 80;
    server_name your-domain.com;

    # 日志
    access_log /var/log/nginx/health-check-access.log;
    error_log /var/log/nginx/health-check-error.log;

    # 最大上传大小
    client_max_body_size 10M;

    # 反向代理到Node.js应用
    location / {
        proxy_pass http://health_check_backend;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection 'upgrade';
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
        proxy_cache_bypass $http_upgrade;

        # 超时设置
        proxy_connect_timeout 60s;
        proxy_send_timeout 60s;
        proxy_read_timeout 60s;
    }

    # 静态文件缓存
    location /_next/static {
        proxy_pass http://health_check_backend;
        proxy_cache_valid 200 365d;
        add_header Cache-Control "public, immutable";
    }
}
```

### 启用配置

```bash
# 创建软链接
ln -s /etc/nginx/sites-available/health-check-admin /etc/nginx/sites-enabled/

# 测试配置
nginx -t

# 重启Nginx
systemctl restart nginx
```

### SSL配置（使用Let's Encrypt）

```bash
# 安装Certbot
sudo apt-get install certbot python3-certbot-nginx

# 获取SSL证书
sudo certbot --nginx -d your-domain.com

# 自动续期
sudo certbot renew --dry-run
```

## 常见问题排查

### 问题1：数据库连接失败

**症状**：启动时报错 "connection refused" 或 "password authentication failed"

**解决方案**：
1. 检查PostgreSQL服务是否运行：
```bash
sudo systemctl status postgresql
```

2. 检查数据库连接配置是否正确

3. 检查防火墙是否开放5432端口

### 问题2：端口被占用

**症状**：启动时报错 "Port 5000 is already in use"

**解决方案**：
```bash
# 查找占用端口的进程
lsof -i :5000

# 或使用ss命令
ss -tlnp | grep :5000

# 杀死进程
kill -9 <PID>
```

### 问题3：依赖安装失败

**症状**：`pnpm install` 报错

**解决方案**：
```bash
# 清除缓存
pnpm store prune

# 删除node_modules和.lock文件
rm -rf node_modules pnpm-lock.yaml

# 重新安装
pnpm install
```

### 问题4：构建失败

**症状**：`pnpm run build` 报错

**解决方案**：
```bash
# 清除构建缓存
rm -rf .next

# 重新构建
pnpm run build
```

### 问题5：页面404

**症状**：访问页面显示404

**解决方案**：
1. 检查路由配置是否正确
2. 检查权限中间件是否拦截
3. 检查Nginx配置是否正确

## 备份与恢复

### 数据库备份

```bash
# 备份数据库
pg_dump -h localhost -U postgres health_check_db > backup_$(date +%Y%m%d).sql

# 恢复数据库
psql -h localhost -U postgres health_check_db < backup_20250204.sql
```

### 自动备份脚本

创建 `/usr/local/bin/backup_db.sh`：

```bash
#!/bin/bash
BACKUP_DIR="/backup/postgresql"
DATE=$(date +%Y%m%d_%H%M%S)
DB_NAME="health_check_db"

# 创建备份目录
mkdir -p $BACKUP_DIR

# 备份数据库
pg_dump -h localhost -U postgres $DB_NAME > $BACKUP_DIR/${DB_NAME}_${DATE}.sql

# 删除30天前的备份
find $BACKUP_DIR -name "*.sql" -mtime +30 -delete

echo "Backup completed: ${DB_NAME}_${DATE}.sql"
```

### 设置定时任务

```bash
# 编辑crontab
crontab -e

# 添加每天凌晨2点备份
0 2 * * * /usr/local/bin/backup_db.sh
```

## 安全加固

### 1. 数据库安全

```sql
-- 修改默认密码
ALTER USER postgres WITH PASSWORD 'strong_password';

-- 删除不需要的数据库
DROP DATABASE IF EXISTS test;

-- 限制远程访问
-- 编辑 pg_hba.conf，只允许本地连接
```

### 2. 应用安全

- 使用强密码
- 启用HTTPS
- 配置CORS
- 实施请求限流

### 3. 服务器安全

```bash
# 更新系统
sudo apt update && sudo apt upgrade -y

# 配置防火墙
sudo ufw enable
sudo ufw allow 22/tcp
sudo ufw allow 80/tcp
sudo ufw allow 443/tcp

# 禁用root登录
sudo sed -i 's/PermitRootLogin yes/PermitRootLogin no/' /etc/ssh/sshd_config
sudo systemctl restart sshd
```
