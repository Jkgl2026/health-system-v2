# 健康自检后台系统 - 生产环境部署标准化指令集

## 📋 部署说明

**适用系统**：Linux（推荐 Ubuntu 20.04+/CentOS 7+）  
**技术栈**：Next.js 14.2.18 + PostgreSQL 14 + Node.js 24+ + pnpm 8+  
**部署方式**：PM2 进程管理 + Nginx 反向代理（可选）

## 🗄️ 三、数据库初始化/数据恢复

### 3.1 创建数据库
```bash
# 切换到 postgres 用户
sudo -u postgres psql

# 创建数据库
CREATE DATABASE health_check_db;

# 创建数据库用户（可选）
CREATE USER health_check_user WITH PASSWORD 'your_secure_password';

# 授予权限
GRANT ALL PRIVILEGES ON DATABASE health_check_db TO health_check_user;

# 退出 PostgreSQL
\q
```

### 3.2 恢复数据库结构
```bash
# 恢复数据库结构
psql -U postgres -d health_check_db -f database/health_check_db_schema.sql

# 或者使用新创建的用户
psql -U health_check_user -d health_check_db -f database/health_check_db_schema.sql
```

### 3.3 恢复数据库数据
```bash
# 恢复初始数据（包含管理员账号和测试用户）
psql -U postgres -d health_check_db -f database/health_check_db_data.sql

# 或者使用新创建的用户
psql -U health_check_user -d health_check_db -f database/health_check_db_data.sql
```

### 3.4 验证数据库恢复
```bash
# 验证管理员账号
psql -U postgres -d health_check_db -c "SELECT * FROM sys_admin;"

# 验证用户数据
psql -U postgres -d health_check_db -c "SELECT COUNT(*) FROM sys_user;"

# 预期结果：
# - 管理员：1 个（admin/123456）
# - 用户：5 个测试用户
```

## 🎯 六、PM2 进程管理启动

### 6.1 安装 PM2
```bash
# 全局安装 PM2
pnpm add -g pm2

# 或者使用 npm
npm install -g pm2
```

### 6.2 创建 PM2 配置文件
```bash
# 创建 ecosystem.config.cjs 文件
cat > ecosystem.config.cjs << 'EOF'
module.exports = {
  apps: [{
    name: 'health-check-system',
    script: 'node_modules/next/dist/bin/next',
    args: 'start',
    cwd: '/opt/health-check-system',
    instances: 1,
    autorestart: true,
    watch: false,
    max_memory_restart: '1G',
    env: {
      NODE_ENV: 'production',
      PORT: 5000
    },
    error_file: '/var/log/health-check-system/error.log',
    out_file: '/var/log/health-check-system/out.log',
    log_date_format: 'YYYY-MM-DD HH:mm:ss Z'
  }]
};
EOF
```

### 6.3 创建日志目录
```bash
# 创建日志目录
sudo mkdir -p /var/log/health-check-system

# 设置日志目录权限
sudo chown -R $(whoami):$(whoami) /var/log/health-check-system
```

### 6.4 启动应用
```bash
# 使用 PM2 启动应用
pm2 start ecosystem.config.cjs

# 查看应用状态
pm2 status

# 预期输出：
# ┌─────┬──────────────────┬─────┬─────┬──────┬──────┐
# │ id  │ name             │ mode│ status│ cpu  │ memory│
# ├─────┼──────────────────┼─────┼─────┼──────┼──────┤
# │ 0   │ health-check-syst│ fork │ online│ 0%   │ 50MB  │
# └─────┴──────────────────┴─────┴─────┴──────┴──────┘
```

### 6.5 配置 PM2 开机自启
```bash
# 保存当前 PM2 进程列表
pm2 save

# 生成开机自启脚本
pm2 startup

# 执行输出的命令（通常是）
# sudo env PATH=$PATH:/usr/bin pm2 startup systemd -u $(whoami) --hp /home/$(whoami)
```

## 🌐 七、Nginx 反向代理配置（可选）

### 7.1 安装 Nginx
```bash
# 安装 Nginx
sudo apt-get update
sudo apt-get install -y nginx

# 启动 Nginx
sudo systemctl start nginx
sudo systemctl enable nginx
```

### 7.2 创建 Nginx 配置文件
```bash
# 创建站点配置文件
sudo cat > /etc/nginx/sites-available/health-check-system << 'EOF'
server {
    listen 80;
    server_name your-domain.com;  # 替换为你的域名或服务器 IP

    # 日志配置
    access_log /var/log/nginx/health-check-system-access.log;
    error_log /var/log/nginx/health-check-system-error.log;

    # 反向代理配置
    location / {
        proxy_pass http://localhost:5000;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection 'upgrade';
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
        proxy_cache_bypass $http_upgrade;
        
        # 超时配置
        proxy_connect_timeout 60s;
        proxy_send_timeout 60s;
        proxy_read_timeout 60s;
    }

    # 静态资源缓存（可选）
    location /_next/static {
        proxy_pass http://localhost:5000;
        proxy_cache_valid 200 60m;
        add_header Cache-Control "public, immutable";
    }
}
EOF
```

### 7.3 启用站点配置
```bash
# 创建软链接
sudo ln -s /etc/nginx/sites-available/health-check-system /etc/nginx/sites-enabled/

# 测试 Nginx 配置
sudo nginx -t

# 重启 Nginx
sudo systemctl restart nginx
```

### 7.4 配置 SSL 证书（推荐）
```bash
# 安装 Certbot
sudo apt-get install -y certbot python3-certbot-nginx

# 自动配置 SSL 证书
sudo certbot --nginx -d your-domain.com

# Certbot 会自动修改 Nginx 配置，启用 HTTPS
```

## 📊 十一、监控和日志

### 11.1 配置日志轮转
```bash
# 创建 logrotate 配置
sudo cat > /etc/logrotate.d/health-check-system << 'EOF'
/var/log/health-check-system/*.log {
    daily
    rotate 7
    compress
    delaycompress
    missingok
    notifempty
    create 0644 $(whoami) $(whoami)
}
EOF

# 测试 logrotate
sudo logrotate -f /etc/logrotate.d/health-check-system
```

### 11.2 配置监控告警（可选）
```bash
# 安装监控工具
pnpm add -g pm2-logrotate

# 初始化 PM2 logrotate
pm2 install pm2-logrotate

# 配置日志轮转
pm2 set pm2-logrotate:max_size 10M
pm2 set pm2-logrotate:retain 7
```
