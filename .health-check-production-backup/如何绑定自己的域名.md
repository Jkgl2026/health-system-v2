#### 推荐购买步骤

**以阿里云为例：**

1. **注册账户**
   - 访问：https://wanwang.aliyun.com/
   - 注册/登录阿里云账户

2. **搜索域名**
   - 在搜索框输入想买的域名（如：`myhealth.com`）
   - 查看是否可用（绿色✅表示可用）
   - 尝试多个备选方案

3. **购买域名**
   - 选择后缀（推荐 `.com`、`.cn`、`.net`）
   - 添加到购物车
   - 支付费用

4. **实名认证**（国内服务商必须）
   - 提交身份证信息
   - 审核通过后域名才能使用

### 方案2：免费域名（不推荐）

| 服务商 | 状态 | 说明 |
|--------|------|------|
| Freenom | ⚠️ 基本不免费 | 曾经提供 .tk/.ml/.ga/.cf/.gq 免费域名 |
| EU.org | ❌ 需要审核 | 需要理由审核 |
| DuckDNS | ⚠️ 仅二级域名 | 只能免费获取 .duckdns.org |

**建议**：免费域名不稳定，建议购买正规域名。

### 域名选择建议

#### 推荐后缀
1. **.com** - 最通用，推荐（69-100元/年）
2. **.cn** - 国内品牌，适合国内用户（30-50元/年）
3. **.net** - 技术类网站，备选（69-100元/年）

#### 命名建议
- **简短易记**：如 `health123.com`
- **包含关键词**：如 `myhealth.com`
- **避免特殊符号**：如 `my-health.com`（不如 `myhealth.com`）
- **中文拼音**：如 `jiankang.com`

#### 预算建议
- 个人项目：100-200元/年（.com）
- 企业项目：500-1000元/年（.com + 保护性注册）

## 如何绑定域名到 Cloudflare Tunnel

### 前提条件

- ✅ 拥有自己的域名
- ✅ 已注册 Cloudflare 免费账户
- ✅ 域名已添加到 Cloudflare

### 配置步骤

#### 第1步：注册 Cloudflare 账户

1. 访问：https://dash.cloudflare.com/sign-up
2. 填写邮箱、密码
3. 验证邮箱
4. 完成注册

#### 第2步：添加域名到 Cloudflare

1. 登录 Cloudflare Dashboard
2. 点击 **"Add a site"**（添加站点）
3. 输入你的域名（如：`myhealth.com`）
4. 选择 **Free**（免费计划）
5. 点击 **Continue**

#### 第3步：修改 DNS 服务器

Cloudflare 会提供两个 DNS 服务器，类似：
```
dina.ns.cloudflare.com
eric.ns.cloudflare.com
```

**在域名服务商处修改 DNS 服务器：**

**以阿里云为例：**

1. 登录阿里云
2. 进入 **域名控制台**
3. 找到你的域名，点击 **"管理"**
4. 点击 **"DNS修改"** → **"修改DNS服务器"**
5. 输入 Cloudflare 提供的两个服务器
6. 等待生效（通常 10分钟 - 24小时）

#### 第4步：等待 DNS 生效

- Cloudflare 会检测 DNS 状态
- 状态变为 **Active** 表示生效
- 通常需要 10分钟 - 24小时

#### 第5步：创建命名隧道

1. 登录 Cloudflare Dashboard
2. 选择你的域名
3. 点击左侧菜单 **Zero Trust** → **Networks** → **Tunnels**
4. 点击 **"Create a tunnel"**
5. 选择 **"Cloudflare Tunnel"**
6. 输入隧道名称（如：`health-system-tunnel`）
7. 点击 **Save tunnel**

#### 第6步：安装 cloudflared（如果还没有）

```bash
# 在服务器上安装 cloudflared
mkdir -p /usr/local/bin
curl -L --output cloudflared https://github.com/cloudflare/cloudflared/releases/latest/download/cloudflared-linux-amd64
chmod +x cloudflared
sudo mv cloudflared /usr/local/bin/cloudflared

# 验证安装
cloudflared --version
```

#### 第7步：获取隧道 Token

1. 在 Cloudflare Tunnel 页面
2. 选择你创建的隧道
3. 点击 **"Configure"**
4. 滚动到 **"Quick Tunnels"** 或 **"One-click SSH"**
5. 或者使用 **"Install and run a connector"** 命令

你会看到类似这样的命令：
```bash
cloudflared tunnel --config /path/to/config.yml run
```

#### 第8步：配置隧道

创建配置文件 `/etc/cloudflared/config.yml`：

```yaml
tunnel: <你的隧道ID>
credentials-file: /etc/cloudflared/<隧道ID>.json

ingress:
  - hostname: myhealth.com
    service: http://localhost:5000
  - hostname: www.myhealth.com
    service: http://localhost:5000
  - service: http_status:404
```

#### 第9步：配置 DNS 记录

1. 在 Cloudflare Dashboard
2. 进入你的域名 → **DNS** → **Records**
3. 点击 **"Add record"**
4. 配置如下：

| 字段 | 值 |
|------|-----|
| Type | **CNAME** |
| Name | **@**（根域名）或 **www** |
| Target | **你的隧道ID**（Cloudflare 会自动填写） |
| Proxy status | ✅ Proxied（橙色云朵） |

5. 保存记录

#### 第10步：启动隧道

```bash
# 使用配置文件启动
cloudflared tunnel --config /etc/cloudflared/config.yml run

# 或者使用隧道 Token 启动
cloudflared tunnel run --token <你的Token>
```

#### 第11步：设置开机自启动

创建 systemd 服务文件 `/etc/systemd/system/cloudflared.service`：

```ini
[Unit]
Description=Cloudflare Tunnel
After=network.target

[Service]
Type=simple
User=root
ExecStart=/usr/local/bin/cloudflared tunnel --config /etc/cloudflared/config.yml run
Restart=on-failure
RestartSec=5s

[Install]
WantedBy=multi-user.target
```

启动服务：

```bash
# 重新加载 systemd
sudo systemctl daemon-reload

# 启动服务
sudo systemctl start cloudflared

# 设置开机自启动
sudo systemctl enable cloudflared

# 查看状态
sudo systemctl status cloudflared

# 查看日志
sudo journalctl -u cloudflared -f
```

#### 第12步：验证访问

访问你的域名：
- `https://myhealth.com/admin/login`
- `https://www.myhealth.com/admin/login`

如果正常显示登录页面，说明配置成功！

## 常见问题

### Q1：需要多长时间生效？

**A**：
- DNS 修改：10分钟 - 24小时
- 通常 30分钟内生效
- 可以使用 `nslookup` 查询

```bash
nslookup myhealth.com
```

### Q2：免费账户有限制吗？

**A**：Cloudflare 免费账户限制：
- ✅ 无限流量
- ✅ 无限隧道
- ✅ 全球 CDN
- ⚠️ 最多 100 条 DNS 记录（个人足够）
- ⚠️ 不支持 WAF 高级规则

### Q3：可以绑定多个域名吗？

**A**：可以！
- 每个域名需要单独添加到 Cloudflare
- 可以在同一个隧道中配置多个域名
- 配置示例：

```yaml
ingress:
  - hostname: myhealth.com
    service: http://localhost:5000
  - hostname: www.myhealth.com
    service: http://localhost:5000
  - hostname: health-site.com
    service: http://localhost:5000
  - service: http_status:404
```

### Q4：如何启用 HTTPS？

**A**：Cloudflare 自动启用 HTTPS！
- ✅ SSL 证书自动签发
- ✅ 自动续期
- ✅ 免费
- 无需任何配置

### Q5：域名到期了怎么办？

**A**：及时续费！
- 到期前会收到邮件提醒
- 可以设置自动续费
- 过期后会被暂停，需要续费

### Q6：可以迁移域名到 Cloudflare 吗？

**A**：可以！
1. 在域名服务商获取授权码（转移密码）
2. 在 Cloudflare 点击 **"Transfer domain"**
3. 输入授权码完成转移

### Q7：域名解析到 Cloudflare 后，还需要域名服务商吗？

**A**：需要！
- 域名服务商只负责域名注册和续费
- DNS 解析由 Cloudflare 处理
- 两者都要续费

### Q8：如何查看 Tunnel 日志？

**A**：
```bash
# 查看 systemd 服务日志
sudo journalctl -u cloudflared -f

# 查看实时日志
tail -f /var/log/cloudflared.log
```

### Q9：Tunnel 断线了怎么办？

**A**：自动重连！
- cloudflared 会自动重连
- systemd 会自动重启服务
- 无需手动干预

### Q10：如何备份隧道配置？

**A**：备份配置文件和凭证文件：
```bash
# 备份配置
cp /etc/cloudflared/config.yml /backup/

# 备份凭证
cp /etc/cloudflared/*.json /backup/

# 恢复
cp /backup/config.yml /etc/cloudflared/
cp /backup/*.json /etc/cloudflared/
```

## 总结

### 拥有域名的步骤
1. 选择域名服务商（阿里云/腾讯云/Namecheap）
2. 搜索并购买域名
3. 完成实名认证（国内服务商）
4. 等待域名生效

### 绑定域名的步骤
1. 注册 Cloudflare 账户
2. 添加域名到 Cloudflare
3. 修改域名 DNS 服务器
4. 等待 DNS 生效
5. 创建命名隧道
6. 配置隧道文件
7. 添加 DNS 记录
8. 启动隧道服务
9. 验证访问

### 优势对比

| 特性 | Quick Tunnel | 命名隧道（自有域名） |
|------|-------------|-------------------|
| 域名 | 随机 | 固定 ✅ |
| HTTPS | 自动 | 自动 ✅ |
| CDN | 自动 | 自动 ✅ |
| 稳定性 | ⭐⭐⭐⭐ | ⭐⭐⭐⭐⭐ |
| 价格 | 免费 | 免费（域名需付费） |
| 难度 | 简单 | 中等 |
| 专业性 | 个人/测试 | 企业/生产 |

### 推荐方案

| 场景 | 推荐方案 |
|------|---------|
| 个人测试 | Quick Tunnel（随机域名） |
| 个人项目 | 命名隧道（自有域名） |
| 企业项目 | 命名隧道（自有域名 + WAF） |
| 生产环境 | 命名隧道（自有域名 + 高级防护） |
