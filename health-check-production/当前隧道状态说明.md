## 两种隧道方式对比

### 方式 1：Quick Tunnel（当前使用）

| 特性 | 说明 |
|------|------|
| **域名** | 随机生成（如：shield-approximate-actions-selective.trycloudflare.com） |
| **稳定性** | ⭐⭐⭐⭐（很稳定） |
| **价格** | 免费 |
| **难度** | 简单（一键启动） |
| **域名变化** | 每次重启可能变化 |
| **适用场景** | 测试、临时使用 |

**优点：**
- ✅ 完全免费
- ✅ 配置简单
- ✅ 自动启动
- ✅ 全球 CDN
- ✅ HTTPS 自动配置

**缺点：**
- ⚠️ 域名随机，不固定
- ⚠️ 重启后域名可能变化

**管理命令：**
```bash
# 查看状态
bash health-check-production/cloudflare-tunnel/manage-tunnel.sh status

# 查看访问地址
bash health-check-production/cloudflare-tunnel/manage-tunnel.sh url

# 重启（会生成新域名）
bash health-check-production/cloudflare-tunnel/manage-tunnel.sh restart
```

---

### 方式 2：命名隧道（自有域名）

| 特性 | 说明 |
|------|------|
| **域名** | 你自己的域名（如：myhealth.com） |
| **稳定性** | ⭐⭐⭐⭐⭐（非常稳定） |
| **价格** | 域名费 69-100元/年，Cloudflare 免费 |
| **难度** | 中等（需要配置） |
| **域名变化** | 固定不变 |
| **适用场景** | 正式项目、长期使用 |

**优点：**
- ✅ 域名固定
- ✅ 更专业
- ✅ 品牌形象好
- ✅ 易于记忆
- ✅ 可以配置多个域名

**缺点：**
- ⚠️ 需要购买域名（69-100元/年）
- ⚠️ 需要配置 DNS
- ⚠️ 配置稍微复杂

**配置步骤：**
1. 购买域名
2. 注册 Cloudflare 账户
3. 添加域名到 Cloudflare
4. 修改 DNS 服务器
5. 创建命名隧道
6. 配置隧道文件
7. 启动服务

**详细文档：**
- `如何绑定自己的域名.md`（完整详细版）
- `如何绑定自己的域名-快速开始.md`（快速上手版）

**辅助脚本：**
```bash
# 运行配置脚本（交互式）
bash health-check-production/cloudflare-tunnel/setup-named-tunnel.sh
```

## 如何选择？

### 使用 Quick Tunnel（当前方式）
如果满足以下条件：
- ✅ 只是测试和临时使用
- ✅ 不介意域名会变化
- ✅ 不想花钱买域名
- ✅ 希望快速搭建

**继续使用当前方式即可！**

### 切换到命名隧道（自有域名）
如果满足以下条件：
- ✅ 这是正式项目
- ✅ 希望有固定的域名
- ✅ 需要长期使用
- ✅ 希望更专业的访问地址

**按照文档配置命名隧道！**
