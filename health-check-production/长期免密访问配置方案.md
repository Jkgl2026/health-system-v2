## ⚠️ 重要说明

### 关于"长期免密访问"

**现状**：即使配置了自定义子域名，**免费版 LocalTunnel 仍然需要每7天输入隧道密码**。这是 LocalTunnel 服务的限制，不是配置问题。

**为什么？**
- LocalTunnel 免费版为了防止滥用，设置了7天密码有效期
- 自定义子域名只是固定了访问地址，不影响密码验证机制
- 每次重启隧道或7天后，都会要求输入密码

## 💡 实现真正"长期免密访问"的方案

### 方案 1：使用 Cloudflare Tunnel（推荐）✅

#### 优势
- ✅ **完全免费**
- ✅ **无需密码**
- ✅ **长期稳定**
- ✅ **HTTPS自动配置**
- ✅ **支持自定义域名**

#### 配置步骤

##### 步骤 1：注册 Cloudflare 账户
1. 访问 https://dash.cloudflare.com/sign-up
2. 注册免费账户
3. 添加一个域名（可以免费注册一个二级域名）

##### 步骤 2：安装 cloudflared 客户端
```bash
# Linux/macOS
curl -L https://github.com/cloudflare/cloudflared/releases/latest/download/cloudflared-linux-amd64 -o /usr/local/bin/cloudflared
chmod +x /usr/local/bin/cloudflared

# 验证安装
cloudflared --version
```

##### 步骤 3：登录 Cloudflare
```bash
cloudflared tunnel login
```
- 会打开浏览器，授权 Cloudflare 访问你的域名

##### 步骤 4：创建隧道
```bash
cloudflared tunnel create health-check-tunnel
```
- 会返回一个隧道 ID，保存这个 ID

##### 步骤 5：配置隧道
创建配置文件 `~/.cloudflared/config.yml`：
```yaml
tunnel: <你的隧道ID>
credentials-file: /root/.cloudflared/<隧道ID>.json

ingress:
  - hostname: health-check.你的域名.com
    service: http://localhost:5000
  - service: http_status:404
```

##### 步骤 6：启动隧道
```bash
cloudflared tunnel run health-check-tunnel
```

##### 步骤 7：设置开机自启
```bash
# Linux systemd 服务
sudo cloudflared service install
sudo systemctl start cloudflared
sudo systemctl enable cloudflared
```

#### 访问地址
```
https://health-check.你的域名.com/admin/login
```

#### 是否需要密码
**❌ 不需要！** Cloudflare Tunnel 完全无需密码，长期稳定访问。

### 方案 2：使用 ngrok 免费版

#### 优势
- ✅ **无需密码**
- ✅ **配置简单**
- ✅ **支持自定义域名（付费）**

#### 配置步骤

##### 步骤 1：注册 ngrok 账户
1. 访问 https://ngrok.com/signup
2. 注册免费账户
3. 获取 authtoken

##### 步骤 2：安装 ngrok
```bash
# Linux
curl -s https://ngrok-agent.s3.amazonaws.com/ngrok.asc | sudo tee /etc/apt/trusted.gpg.d/ngrok.asc >/dev/null
echo "deb https://ngrok-agent.s3.amazonaws.com buster main" | sudo tee /etc/apt/sources.list.d/ngrok.list
sudo apt update && sudo apt install ngrok

# 验证安装
ngrok version
```

##### 步骤 3：认证
```bash
ngrok config add-authtoken <你的authtoken>
```

##### 步骤 4：启动隧道
```bash
ngrok http 5000
```

#### 访问地址
```
https://<随机子域名>.ngrok-free.app/admin/login
```

#### 是否需要密码
**❌ 不需要！** ngrok 免费版无需密码。

### 方案 3：使用 frp（需要服务器）

#### 优势
- ✅ **完全免费**
- ✅ **无需密码**
- ✅ **自定义域名**
- ✅ **长期稳定**

#### 配置步骤

##### 步骤 1：准备服务器
- 需要一台有公网IP的服务器（可以是VPS）
- 安装 frp 服务端

##### 步骤 2：配置服务端（服务器端）
创建 `frps.ini`：
```ini
[common]
bind_port = 7000
vhost_http_port = 80
vhost_https_port = 443
```

##### 步骤 3：启动服务端
```bash
./frps -c frps.ini
```

##### 步骤 4：配置客户端（沙箱端）
创建 `frpc.ini`：
```ini
[common]
server_addr = <服务器IP>
server_port = 7000

[health-check]
type = http
local_port = 5000
custom_domains = health-check.你的域名.com
```

##### 步骤 5：启动客户端
```bash
./frpc -c frpc.ini
```

#### 是否需要密码
**❌ 不需要！** 但需要配置服务器。

### 方案 4：使用 SSH 反向代理（需要服务器）

#### 优势
- ✅ **完全免费**
- ✅ **无需密码（可配置免密登录）**
- ✅ **长期稳定**

#### 配置步骤

##### 步骤 1：配置免密登录（沙箱端）
```bash
# 生成密钥对
ssh-keygen -t rsa

# 复制公钥到服务器
ssh-copy-id root@<服务器IP>
```

##### 步骤 2：创建 SSH 隧道
```bash
ssh -N -R 80:localhost:5000 root@<服务器IP>
```

##### 步骤 3：配置反向代理（服务器端）
使用 Nginx 配置：
```nginx
server {
    listen 80;
    server_name health-check.你的域名.com;

    location / {
        proxy_pass http://localhost:80;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
    }
}
```

#### 是否需要密码
**❌ 不需要！** 配置免密登录后无需密码。
