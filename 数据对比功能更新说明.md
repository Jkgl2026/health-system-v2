# 数据对比功能更新说明

## 更新时间
2025-02-04

## 更新内容

### ✅ 已修复的问题

#### 问题1：数据对比功能使用模拟数据
**问题描述：** 之前的数据对比功能使用的是模拟数据（`mockCompareData.ts`），无法显示用户真实填写的数据。

**解决方案：**
1. 创建了新的API接口：`/api/admin/users/history`，用于获取用户的真实历史记录
2. 修改了数据对比页面（`src/app/admin/compare/page.tsx`），改为调用真实数据API
3. 删除了模拟数据依赖

**影响：**
- ✅ 现在可以查询用户真实填写的数据
- ✅ 几分钟前填写的数据可以立即看到
- ✅ 支持通过手机号或姓名查询用户历史记录
- ✅ 数据对比功能使用真实数据库数据

#### 问题2：1月28日的全部数据对比功能
**问题描述：** 用户反馈数据对比功能不完整，某些功能缺失。

**解决方案：**
- 数据对比功能包含所有1月28日版本的功能：
  - ✅ 基本信息对比（姓名、年龄、性别、职业）
  - ✅ BMI对比（身高、体重、BMI值）及变化分析
  - ✅ 血压对比及变化分析
  - ✅ 健康要素对比（8个指标：气血、循环、毒素、血脂、寒凉、免疫力、情绪、整体健康）
  - ✅ 症状检查对比（症状数量、总分）
  - ✅ 用户选择对比（选择的方案）
  - ✅ 完成情况对比（各项任务完成状态）

**技术实现：**
- 使用固定定位的简单弹窗，确保所有浏览器都能正常显示
- 详细的调试日志，方便排查问题
- 完整的错误处理和用户反馈
- 最多支持3个版本的对比

## API 接口说明

### 1. 获取用户历史记录
**接口：** `GET /api/admin/users/history`

**参数：**
- `phone`（可选）：手机号
- `name`（可选）：姓名

**返回：**
```json
{
  "success": true,
  "users": [
    {
      "id": "user-id-1",
      "name": "张三",
      "phone": "13800138001",
      "age": 35,
      "gender": "男",
      "createdAt": "2025-01-10T08:30:00.000Z",
      "isLatestVersion": true,
      ...
    }
  ]
}
```

**说明：**
- 通过手机号查询时，会返回该手机号的所有历史记录（通过 `phoneGroupId` 关联）
- 通过姓名查询时，会返回所有匹配姓名的用户记录
- 结果按创建时间倒序排列

### 2. 获取用户完整数据
**接口：** `GET /api/admin/users/[userId]`

**返回：**
```json
{
  "success": true,
  "data": {
    "user": {
      "id": "user-id-1",
      "name": "张三",
      "symptomChecks": [...],
      "healthAnalysis": [...],
      "userChoices": [...],
      "requirements": {...}
    }
  }
}
```

**说明：**
- 返回用户的完整数据，包括基本信息、症状检查、健康分析、用户选择、完成情况
- 用于数据对比功能

## 使用方法

### 管理员使用步骤

1. **登录管理后台**
   - 访问：`/admin/login`
   - 输入账号：`admin`
   - 输入密码：`admin123`

2. **打开数据对比页面**
   - 访问：`/admin/compare`

3. **查询用户历史记录**
   - 输入手机号或姓名
   - 点击"查询历史记录"
   - 系统会显示该用户的所有历史记录

4. **选择要对比的版本**
   - 勾选你想对比的记录（最多3条）
   - 点击"开始对比"

5. **查看对比结果**
   - 基本信息对比
   - BMI对比及变化分析
   - 血压对比及变化分析
   - 健康要素对比及变化分析
   - 症状检查对比
   - 用户选择对比
   - 完成情况对比

## 注意事项

1. **数据实时性**
   - 用户填写数据后，管理后台可以立即看到
   - 无需刷新页面或等待

2. **历史记录管理**
   - 同一用户多次填写会创建多条记录
   - 系统自动标记最新版本（`isLatestVersion`）
   - 通过 `phoneGroupId` 关联同一用户的所有记录

3. **对比限制**
   - 最多支持对比3个版本
   - 至少需要选择2个版本才能对比

4. **数据安全**
   - 所有API都需要管理员认证
   - 只有管理员可以查看用户数据

## 故障排查

### 问题：查询不到用户历史记录
**可能原因：**
1. 手机号或姓名输入错误
2. 用户没有填写过数据
3. 数据库连接问题

**解决方案：**
1. 检查手机号或姓名是否正确
2. 确认用户已经在用户端填写过数据
3. 查看浏览器控制台是否有错误信息

### 问题：对比结果为空
**可能原因：**
1. 选择的用户ID不正确
2. 用户数据不完整
3. API调用失败

**解决方案：**
1. 确认选择的记录是有效的用户记录
2. 检查浏览器控制台的错误信息
3. 确认用户已经完成了症状检查和健康分析

### 问题：页面加载缓慢
**可能原因：**
1. 数据量过大
2. 网络连接问题

**解决方案：**
1. 减少对比的版本数量
2. 检查网络连接
3. 查看服务器日志
